# DDALAB Component Library

Welcome to the DDALAB component documentation. This Storybook contains all the reusable UI components used in the DDALAB desktop application.

## Overview

DDALAB (Delay Differential Analysis Laboratory) is a scientific computing application for performing Delay Differential Analysis on EDF and ASCII files. The UI is built with:

- **React 19** - UI framework
- **Next.js 16** - Full-stack framework with Turbopack
- **Tauri v2** - Desktop application framework (Rust backend)
- **Tailwind CSS** - Utility-first styling
- **Radix UI** - Accessible component primitives
- **shadcn/ui** - Component design system

## Component Categories

### UI Components

Core building blocks for the interface:

- **Button** - Primary interaction element with multiple variants
- **Input** - Form input with validation states
- **Badge** - Status indicators and labels
- **Card** - Content containers
- **Progress** - Progress indicators
- **Tabs** - Content organization
- **Switch** - Boolean toggles

### Feature Components

DDALAB-specific components (documented separately):

- **FileManager** - File browser and management
- **DDAAnalysis** - Analysis configuration panel
- **DDAResults** - Results visualization
- **TimeSeriesPlot** - EEG data visualization
- **SettingsPanel** - Application settings

## Usage

Import components from the `@/components/ui` path:

```tsx
import { Button } from "@/components/ui/button";
import { Card, CardHeader, CardContent } from "@/components/ui/card";
import { Input } from "@/components/ui/input";

function Example() {
  return (
    <Card>
      <CardHeader>Example Form</CardHeader>
      <CardContent>
        <Input placeholder="Enter value..." />
        <Button>Submit</Button>
      </CardContent>
    </Card>
  );
}
```

## Accessibility

All components are built with accessibility in mind:

- Proper ARIA attributes
- Keyboard navigation support
- Focus management
- Screen reader compatibility

Components use Radix UI primitives which handle most accessibility concerns automatically.

## Contributing

When adding new components:

1. Create the component in `src/components/ui/`
2. Add stories in `src/stories/ui/ComponentName.stories.tsx`
3. Include all variants and states
4. Add DDALAB-specific examples where relevant
